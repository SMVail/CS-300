What was the problem you were solving in the projects for this course?
- The problem being solved was designing a program that allows efficient access, organization, and display of course information. Choosing the most effective data structure to support operations like searching, sorting, and printing course information was the most challenging aspect. It required building multiple implementations using a vector, hash table, and binary search tree, and then analyzing their performance.

How did you approach the problem? Consider why data structures are important to understand.
- I focused on understanding how different data structures impact efficiency and usability. Data structures are important because they directly affect how quickly data can be accessed and manipulated. By implementing each structure, I was able to compare their strengths and weaknesses and make a more informed decision about which one best fit the needs of this program.

How did you overcome any roadblocks you encountered while going through the activities or project?
- One of the biggest challenges I encountered was proper file input and ensuring that prerequisite data was valid. The two-pass file reading approach was the most confusing, but I overcame this by breaking it down into steps, first collecting all course numbers, then validating prerequisites during the second pass. I also faced issues with tree insertion logic and debugging syntax errors (I think because I use Visual Studio and there was a .Json processing error), which I resolved by tracing the logic and testing smaller sections of code.

How has your work on this project expanded your approach to designing software and developing programs?
- This project expanded my approach to software design by showing me the importance of planning before coding. Writing pseudocode first helped familiarize my understanding of expectations, organize my thoughts, decrease redundancy, and identify potential issues early. It also reinforced the idea that choosing the right data structure is just as important as writing correct code, since it can significantly impact performance and scalability.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
- My work throughout this project improved the way I write programs, especially pseudocode, by encouraging me to focus on readability, structure, and maintainability. I now pay more attention to early planning, modular functions, and consistent formatting. I also think more about how future changes might affect my code, which helps me write programs that are easier to update and adapt. These skills will be valuable in future coursework and real-world programming tasks.
